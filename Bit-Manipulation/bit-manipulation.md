# Bit manipulation

## Basics

### 原码、补码

在计算机中，十进制 -1 与 1 的相加是通过补码加法器直接完成的。计算机不对正负号做特殊逻辑区分，而是将符号位视为数值位的一部分参与运算。 以常见的 8 位二进制（1 Byte） 为例，运算过程如下： 
1. 将数值转换为补码 
- 1 的补码：
    - 正数的补码与其原码相同。
    - 1 的原码是 0000 0001，其补码即为 0000 0001。
- -1 的补码：
    - 负数的补码计算步骤：原码 ➡️ 反码（除符号位外按位取反）➡️ 补码（反码 + 1）。
    1. -1 的原码：1000 0001
    1. -1 的反码：1111 1110
    1. -1 的补码：1111 1110 + 1 =  1111 1111。 
2. 进行二进制加法 
计算机将这两个补码送入加法器直接相加： 
```
      1111 1111  (-1 的补码)
    + 0000 0001  ( 1 的补码)
    -----------
    (1)0000 0000
```

### 二进制计算、溢出Overflowing

All binary numbers invovled in calculation are considered as 补码 (Two's complement)

Overflowing on 4-bit data type

5 的二进制： 0101

6 的二进制： 0110

相加： 0101 + 0110 = 1011

**以上看到的二进制数都被视为补码**

❗❗重点来了： 在有符号位系统里，首位是 1 表示负数。补码1011 代表 -5。

（反推原码方法：取反加一，即 0100 + 1 = 0101，也就是 5 的负数）

（补码转换本身是自反的--self-inverse, 即重复相同的运算就会自我抵消）

也可以尝试反推原码：
- 补码： 1011
- 减1： 1010
- 取反（符号位不变）： 1101 ➡️ -5（原码表示）。
